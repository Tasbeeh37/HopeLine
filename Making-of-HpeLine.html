Making Of HopeLine

The website HopeLine was designed as an educational and informational mental-health support platform aimed at teenagers and young adults. The main objective of the website is to provide users with trustworthy mental-health information, emotional encouragement, and access to supportive resources in a safe and friendly environment. As part of the design process, low-fidelity mock-ups were created using a design tool rather than HTML. These mock-ups were used to plan the layout, navigation structure, and content organization before the development stage began. This approach helped the group visualize the website structure, identify potential design issues early, and ensure a consistent user experience across all pages.

Website Color Palette

Color Design Justification

These colors directly influence how users emotionally interact with the website. Using soft blue tones reduces anxiety, while yellow introduces optimism, helping teenagers feel safe and encouraged.

  • Light Blue (#B3E5FC): Represents calmness, peace, and emotional comfort. It helps users feel relaxed when visiting the website.
  • Dark Blue (#0056B3 / #0277BD): Symbolizes trust, confidence, and reliability. This color is mainly used in navigation bars and titles to build professional credibility.
  • Soft Yellow (#FFD54F): Represents hope, warmth, and positivity. It is used as an accent color to give emotional encouragement.

This combination was chosen to avoid harsh colors and ensure that users feel emotionally safe and supported while browsing the website.

Mock-up 2: About Depression Page

Design Justification

The page layout is simple and easy to read. Clear headings divide the content into sections to avoid overwhelming the users with too much information.

Code Justification

Mock-up 4: Booking Page

Design Justification

The first mock-up shows the available appointments in a table to help users choose easily. The second mock-up focuses on a simple form layout for confirming the booking. Clear labels and spacing reduce confusion and make the process user-friendly.

Code Justification

HTML forms structure the booking process, while CSS ensures a consistent layout. JavaScript handles validation and confirmation alerts. This organized structure makes future updates easy and allows reuse of form components.

Mock-up 3: Help YourSelf Page

Design Justification

Card-style layouts with icons make self-care activities visually easier to follow and more engaging for users.

Code Justification

Bootstrap cards help with layout consistency and responsiveness. CSS hover effects add interaction feedback. Maintainability is improved by separating structure, styling, and logic.

Mock-up 6: Making Of HopeLine

Design Justification

This page clearly explains the design decisions and development process. The layout is simple so users can focus on the documentation.

Code Justification

Semantic HTML and reusable CSS classes improve maintainability. The structure allows easy updates and future page additions.

Mock-up 5: Our Story Page

Design Justification

The page uses a storytelling layout to guide the reader through the journey of the project from beginning to end. Large headings and separated sections make each part easy to read. A calm color theme supports emotional connection and keeps the page comfortable for long reading.

Code Justification

HTML sectioning elements are used to divide the content into logical parts such as introduction, challenges, learning, and conclusion. CSS controls text size, spacing, and alignment to improve readability. Reusable classes and consistent structure help maintain the page and make updates easy.
